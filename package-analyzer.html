Case Study: Package Analyzer

The Problem

Open Source packages are increasingly used to build modern software. Licenses can conflict with one another, and as such it is important to understand package composition from a licensing perspective. Enter Package Analyzer &emdash; a tool designed to answer the question, "What licenses are present in my product?"

Input: package.json

Output: Visualization of metadata, license(s), version(s), and links for further exploration as needed.



The Process

The initial plan was to employ a textarea for JSON input. This would allow users to paste in their package.json file and see the results. However, this approach presented a couple of UX challenges during the testing phase:

  • The friction of having to open the file and manually select the contents is unnecessary.
  • The JSON is sometimes large and complex, and there was a tendency to become distracted by the complexity and formatting, when the content of the file is not the focus.

To address these concerns, I implemented a file-input approach that allows users to more simply select a package.json file for upload. This approach is faster and more intuitive.


Including small touches like basic error handling, a count of dependencies and column sorting where it makes sense based on workflow (by Author, by License, by Name) enhances the user experience.
